Points

Find the simplest algebraic curve through a set of points

Live Demo Source

Click a few points on a grid and instantly see the simplest algebraic curve that passes through all of them. Not a polynomial fit or a regression — an exact implicit equation with small integer coefficients, like x² + y² = 25 or x · y = 6. The solver finds lines, conics, cubics, and quartics in under a millisecond.

Null spaces and Vandermonde matrices

The algorithm treats curve-finding as a linear algebra problem. For a given degree d, there is a finite set of monomial terms — degree 2, for example, has six: 1, x, y, x², xy, y². Any implicit curve of that degree is a linear combination of these monomials set equal to zero. The solver builds a Vandermonde matrix where each row is a clicked point and each column is a monomial evaluated at that point, then looks for a null vector — a set of coefficients that makes every row vanish. That null vector is the equation of the curve.

To favor simple results, the solver searches sparse subsets first. For each degree (1 through 4), it tries all k-subsets of monomial columns starting from k = 2 and working upward. The first non-trivial null vector found this way uses the fewest terms possible, which tends to produce the most elegant equation. When multiple solutions exist at the same sparsity, a scoring function picks the one with the smallest coefficients and the most symmetric use of x and y — preferring x² + y² = 25 over an equivalent but lopsided form.

From floats to exact integers

Gaussian elimination with partial pivoting produces floating-point null vectors. These need to be converted to exact integer coefficients to display a clean equation. The solver uses continued fraction expansion to rationalize each coefficient, caps denominators at 1000, then multiplies through by the LCM of all denominators and reduces by the GCD. The result is verified using i128 integer arithmetic — no floating-point at all — to confirm that the equation evaluates to exactly zero at every input point. If verification fails, the candidate is discarded and the search continues.

Rendering the curve

Once an equation is found, the grid draws the full curve as an overlay using a marching squares algorithm. It evaluates the implicit function at the corners of a fine grid of cells, detects sign changes that indicate the curve crosses through a cell, and interpolates the crossing points along each edge. The result is a smooth contour that updates instantly as you add or remove points. The canvas handles hi-DPI displays and points snap to a half-unit grid for cleaner placement.

Running in the browser

The solver is Rust compiled to WebAssembly, called from a Web Worker with 30ms debouncing so the UI stays responsive. Click anywhere on the grid to place a point; click an existing point to remove it. The equation and its curve overlay appear in real time. The entire solver is around 500 lines of Rust with no dependencies beyond wasm-bindgen.
